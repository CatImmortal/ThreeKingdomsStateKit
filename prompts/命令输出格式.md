# 命令输出格式

## 一、核心执行机制与红线约束
- **触发条件**: 仅在状态确有变化时，于正文最末尾输出。若无变化，绝不输出 `UpdateVariable`。
- **格式严格性**: 命令块固定包裹于 `<UpdateVariable>...<Commands>...</Commands></UpdateVariable>` 内。`Commands` 必须是纯粹、合法的 JSON 数组，**严禁任何注释、解释或 Markdown 代码块(如 ```json)**。多条命令按数组顺序执行。
- **增量更新**: 只写本轮正文中明确提及或必定引发变化的字段，未提及字段**绝对不要补全**！
- **命名锚定**: 绝对不允许改名！`data.名称` / `changes.名称` 必须与定位用 `名称` 一致。`势力.主公` 必须是具体 NPC 姓名（必须已存在/本轮创建，禁称号/职位/泛指）。需改名只能删旧建新。

## 二、标签模板
${正常叙事正文}
<UpdateVariable>
<Analysis>
- 经过时间: [距上次经过了多久]
- 场景类型: [日常/社交/检定/战斗/经营/军队战/军团战]
- 更新分析: [150词内，基于正文精准分析需增/删/改的数据]
  </Analysis>
  <Commands>
  [
  { "type": "UpdateWorld", "changes": { "当前时间": { "月": 4 } } }
  ]
  </Commands>
  </UpdateVariable>

## 四、共享子结构
世界时间: { 年?: number, 月?: number, 日?: number, 时辰?: string }
地点结构: { 州?: string, 城?: string, 地点?: string }
世界事件输入: { 事件名: string, 简述: string, 日期: 世界时间 }
六维基础字段: { 武力?: number, 智力?: number, 统率?: number, 政治?: number, 魅力?: number, 体质?: number }
装备条目: { 伤害骰?: string, 先攻加值?: number, 攻击加值?: number, 防御DC加值?: number, 伤害减免?: number, 其他效果?: string }
物品条目: { 名称: string, 品质: 物品品质, 类型: 物品类型, 描述: string, 装备条目?: 装备条目 }
装备栏: { 主武器?: 物品条目, 副武器?: 物品条目, 护甲?: 物品条目, 坐骑?: 物品条目, 饰品1?: 物品条目, 饰品2?: 物品条目, 饰品3?: 物品条目 }
- `伤害骰/先攻加值/攻击加值/防御DC加值/伤害减免/其他效果` 只能写在 `装备条目` 内
- 禁止写成 `主武器.伤害骰`，正确路径是 `主武器.装备条目.伤害骰`

状态记录: Record<string, { 效果?: string, 剩余?: string }>

## 五、命令白名单与业务约束
注: JSON结构中未标 `?` 的参数为必填。所有 Commands 按数组顺序执行。

[系统/资源类]
- UpdateWorld: { type: "UpdateWorld", changes: { 当前时间?, 当前地点?: 地点结构, 当前剧本?, 天气?, 近期事件?: 世界事件输入[] } } // 约束: 追加单条事件优先用 AppendRecentEvent
- AppendRecentEvent: { type: "AppendRecentEvent", event: 世界事件输入 }
- UpdatePlayerBase: { type: "UpdatePlayerBase", changes: { 六维?: 六维基础字段, 战斗数据?: 战斗数据基础字段, 物品栏?: Record<string, 物品栏条目>, 所属势力?, 兵种适性?: 兵种适性表, 声望?, 金钱?, 积分?, 官职?, 爵位?, 后宫和谐度?: number } } // 约束: 小额资源增减优先用 AdjustPlayerResource。战斗数据.特殊动作仅在由装备/专长明确解锁时写入。移除物品请将对应 `物品栏[物品名].数量` 设为 0。
- AdjustPlayerResource: { type: "AdjustPlayerResource", mode: string, changes: { 当前生命值?, 当前体力值?, 声望?, 金钱?, 积分?, 后宫和谐度?: number } }

[势力/城建类]
- UpsertFaction: { type: "UpsertFaction", 名称: string, changes: { 名称?, 主公?, 规模?, 正统性?, 情报网?, 金钱?, 粮草?: number } } // 约束: 绝对不要整块重写城池/军队/外交/政策对象。
- UpsertCity: { type: "UpsertCity", 势力名称: string, 名称: string, data: 城池 }
- AddCityFacility / RemoveCityFacility: { type: "命令名", 势力名称: string, 名称: string, facility: string }
- RemoveCity: { type: "RemoveCity", 势力名称: string, 名称: string }
- UpdateDiplomacy: { type: "UpdateDiplomacy", 名称: string, changes: Record<string, number> } // 约束: changes键名直接写目标势力名称
- UpdatePolicy: { type: "UpdatePolicy", 名称: string, changes: 政策 }

[军事/战役类]
- UpsertArmy: { type: "UpsertArmy", 势力名称: string, 名称: string, data: 军队 } // 约束: 特色军名写在 名称。
- RemoveArmy: { type: "RemoveArmy", 势力名称: string, 名称: string }
- CreateArmyBattle: { type: "CreateArmyBattle", 名称: string, data: { 类型: 军队战类型, 地形: string, 攻方势力: string, 守方势力: string, 攻方军队: string[], 守方军队: string[], 攻城数据?: 攻城数据 } } // 约束: 同一时间仅存一场。攻城战须带攻城数据。
- UpdateArmyBattle: { type: "UpdateArmyBattle", 名称: string, changes?: { 阶段?, 遭遇战当前轮次?, 攻城战当前天数?, 攻城数据? }, 攻方军队?: Record<string, 参战军队引用>, 守方军队?: Record<string, 参战军队引用> } // 约束: 名称须与当前对象一致。

[角色/社交类]
- UpsertNpc: { type: "UpsertNpc", 名称: string, data: { 名称?, 品质?, 所属势力?, 定位?, 所在地?: 地点结构, 好感?, 简述?, 羁绊?: Record<string, string>, 六维?: 六维基础字段, 战斗数据?: 战斗数据基础字段, 武将信息?: 武将信息, 美人属性?: 美人属性 } }
  // 业务约束: 仅生成有明确互动的NPC，禁填背景板。出现时尽量填所在地。首次创建互动NPC必须提供完整六维。战斗/武将/美人属性等深层结构仅在涉及时补全。
- UpdateNpcRelation: { type: "UpdateNpcRelation", 名称: string, mode: string, 好感?: number, 羁绊?: Record<string, string> }
- RemoveNpc: { type: "RemoveNpc", 名称: string }

[任务/商业类]
- UpsertQuest: { type: "UpsertQuest", 名称: string, data: { 名称?, 类型?, 状态?, 时限?, 目标?: Record<string, 任务目标更新> } }
- UpdateQuestState: { type: "UpdateQuestState", 名称: string, 状态: 任务状态, 目标?: Record<string, 任务目标更新> }
  // 业务约束: 只推状态，禁重写全任务。主目标完成显式推至"可提交"。仅用户明示"交付/回报"才转"已完成"并发放奖励。
- RemoveQuest: { type: "RemoveQuest", 名称: string }
- UpsertShopItem: { type: "UpsertShopItem", 名称: string, data: { 物品?: 物品条目, 分类?: 商品分类, 价格?: number } }
- RemoveShopItem: { type: "RemoveShopItem", 名称: string }
